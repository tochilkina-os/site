# План: автономный дубликат сайта без Tailwind CDN

## Цель

Создать локальную копию текущего сайта, которая визуально повторяет `index.html`, но не требует доступа к `https://cdn.tailwindcss.com` и корректно открывается локально.

## Область работ

### Входит в scope

- Создать отдельную автономную HTML-страницу, например `index-local.html` или `index-standalone.html`.
- Добавить локальный CSS-файл, например `css/site.css`.
- При необходимости добавить локальный JavaScript-файл, например `js/site.js`.
- Сохранить структуру текущего сайта:
  - шапка;
  - hero-блок;
  - философия;
  - самообразование;
  - достижения;
  - мониторинг;
  - методика;
  - блок для родителей;
  - контакты.
- Заменить Tailwind-классы на обычные CSS-классы.
- Сохранить работу локального изображения `./img/photo_main.jpg`.
- Сохранить интерактивность блока мониторинга и анимации появления.

### Не входит в scope

- Не изменять оригинальный `index.html`, чтобы оставить текущую CDN-версию как резервную.
- Не подключать Tailwind локально через сборщик, если нужна простая автономная HTML/CSS-версия.
- Не внедрять сложную сборку через npm/Vite/Webpack без отдельного решения.

## Предлагаемые файлы

- `index-local.html` — автономная копия страницы.
- `css/site.css` — ручные стили вместо Tailwind.
- `js/site.js` — локальный JavaScript для интерактивности, если код будет вынесен из HTML.

## План реализации

1. Создать автономную HTML-копию
   - Скопировать структуру текущего `index.html`.
   - Удалить подключение `https://cdn.tailwindcss.com`.
   - Подключить локальный CSS: `./css/site.css`.
   - Заменить путь к локальному фото на POSIX-формат: `./img/photo_main.jpg`.

2. Переписать классы
   - Заменить utility-классы Tailwind на семантические CSS-классы:
     - `site-header`;
     - `container`;
     - `section`;
     - `hero`;
     - `hero-grid`;
     - `button` / `button-primary`;
     - `card`;
     - `nav-link`;
     - `stats-card`;
     - `progress-bar`.

3. Реализовать CSS вручную
   - Цветовая палитра:
     - paper: `#fdfbf7`;
     - ink: `#4a433e`;
     - terracotta: `#d98a6c`;
     - sage: `#a3b19b`;
     - sun: `#e6c27a`;
     - clay: `#e2d7ce`.
   - Настроить:
     - базовую типографику;
     - адаптивные контейнеры;
     - сетки;
     - карточки;
     - кнопки;
     - декоративные фоновые формы;
     - мягкие тени;
     - glass-header;
     - progress bars;
     - reveal-анимации.

4. Убрать или локализовать внешние зависимости
   - Удалить Tailwind CDN.
   - Заменить Lucide CDN на inline SVG или текстовые/простые CSS-иконки.
   - Для Google Fonts выбрать один из вариантов:
     - оставить как необязательное улучшение;
     - заменить системными шрифтами для полной автономности;
     - скачать шрифты локально и подключить через `@font-face`.
   - Внешние Unsplash/texture URL заменить локальными изображениями, CSS-градиентами или декоративными блоками.

5. Сохранить JavaScript-функции
   - Перенести или оставить локально код для:
     - переключения годов в «Мониторинге»;
     - анимации появления блоков при скролле;
     - изменения шапки при скролле.
   - Удалить `lucide.createIcons();`, если Lucide больше не используется.

6. Проверить результат
   - Запустить локальный сервер:
     ```powershell
     python -m http.server 8000
     ```
   - Открыть:
     ```text
     http://localhost:8000/index-local.html
     ```
   - Дополнительно проверить прямое открытие через `file:///`.

## Критерии готовности

- Сайт отображается с разметкой и стилями без доступа к Tailwind CDN.
- Основная локальная фотография отображается.
- Навигация по якорям работает.
- Блок «Мониторинг» переключает годы и обновляет значения.
- Анимации появления работают или аккуратно деградируют.
- На мобильной ширине сайт остаётся читаемым.
- В консоли браузера нет ошибок из-за отсутствующих CDN-скриптов.

## Риски и смягчение

- Риск: полное визуальное совпадение с Tailwind-версией потребует ручной настройки CSS.
  - Смягчение: сначала сделать визуально близкую MVP-копию, затем точечно улучшить отступы, сетки и карточки.
- Риск: внешние изображения не будут доступны без интернета.
  - Смягчение: заменить их локальными картинками или CSS-декором.
- Риск: Google Fonts недоступны.
  - Смягчение: использовать системные fallback-шрифты или локальные файлы шрифтов.

## Рекомендуемый подход

Сделать отдельную автономную версию `index-local.html`, не изменяя оригинальный `index.html`. Это безопасно: текущая CDN-версия останется как резервная, а локальная версия будет работать без Tailwind.
